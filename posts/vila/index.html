cho A = [1..100]

với mỗi n thuộc A:
	nếu n % 3 == 1:
		xuất n, "chia 3 dư 1"
	còn nếu n % 3 == 2:
		xuất n, "chia 3 dư 2"
	còn không:
		xuất n, "chia hết cho 3"

Tính hàm hợp, với \((f.g)(x) = f(g(x))\)

cho f(x) = 2x(x^2 - 2x)(3x - 5)
cho g(x) = 5x

cho a = 5
xuất f.g(a)
xuat 1

2x(x^2 - 2x)(3x - 5) là cách viết ngắn gọn hơn của 2*x*(x^2 - 2*x)*(3*x - 5)

Sử dụng list comprehension để định nghĩa mảng vô hạn các phần tử, với fib[i] là phần tử thứ i trong dãy fibonacci

cho fib = {0, 1, 1} + { fib[n-1] + fib[n-2] | n thuộc [3..] }

với mỗi n thuộc fib:
    xuất n

List comprehension có tính “lazy”, tức là khi ta định nghĩa fib thì chưa phần tử nào trong mảng được tính cả, chỉ khi ta cần dùng tới phần tử nào thì phần tử đó mới được tính ra, do đó ta có thể dễ dàng tạo ra mảng vô hạn phần tử trong Vila

Khai báo biến

Biến được định nghĩa thông qua từ khóa cho, với cấu trúc như sau

cho <tên biến> = <biểu thức>

Ví dụ

cho x = 10
cho ngày sinh = 15

Một số hằng số được định nghĩa sẵn

xuất Pi // 3.14159...
xuất E  // 2.71828...

Khai báo hàm

Khai báo hàm tương đối giống với khai báo biến

cho <tên hàm>(<danh sách các tham số>) = <biểu thức>

Ví dụ

cho f(x,y,z) = 2x(x^2 - 2x + 1) - y^z
xuất f(100, 5, 3)

Ở ví dụ trên, hàm chỉ được định nghĩa ngắn gọn trong một hàng, để viết một số hàm phức tạp, cần thực hiện nhiều câu lệnh hơn, ta có thể dùng “Biểu thức nhóm”

cho f(x) = (cho a = 5; cho b = 6; a+b+x)
xuất f(2) // 13

Biểu thức cuối cùng trong biểu thức nhóm sẽ là giá trị trả về cho toàn bộ biểu thức, ở ví dụ trên a+b+x = 5+6+2 = 13 là giá trị của f(2)

Trên thực tế ta sẽ tách các câu lệnh trong biểu thức nhóm bằng cách xuống dòng

// Trả về đúng nếu n là số chẵn, sai nếu n là số lẻ
cho là chẵn(n) = (
	cho two = 2
	nếu n % two == 0:
		=> đúng
	sai
)

Dấu suy ra => có ý nghĩa tương đương với câu lệnh return ở những ngôn ngữ khác, nếu gặp => thì biểu thức nhóm sẽ kết thúc và lấy biểu thức sau => làm giá trị trả về

xuất sin(Pi)  // 0
